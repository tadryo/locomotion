# Go2 鬼ごっこ (Tag Game)

Go2ロボットが追跡者（鬼）から逃げることを学習する強化学習環境です。

## 概要

この環境では：
- **逃走者ロボット**（学習対象）: PPOで訓練され、鬼から逃げることを学習
- **追跡者ロボット**（鬼）: ルールベースの制御で逃走者を追跡
- **目標**: 逃走者が20秒間（1000ステップ）鬼に捕まらずに生き延びること

## 主な特徴

### 環境設定 (`go2_tag_env.py`)

1. **2体のロボット**:
   - 逃走者: 初期位置 `(0, 0, 0.42)`、ランダムな位置に配置
   - 追跡者: 初期位置 `(-3, 0, 0.42)`、逃走者から2-3m離れた位置に配置

2. **観測空間** (45次元):
   - 角速度 (3)
   - 重力投影 (3)
   - **追跡者の相対位置** (3) ← 新規追加
   - 関節角度 (12)
   - 関節速度 (12)
   - アクション (12)

3. **報酬関数**:
   - `avoid_chaser` (5.0): 追跡者から離れることへの報酬
   - `forward_vel` (1.0): 前進速度への報酬
   - `survival` (0.5): 生存報酬
   - `lin_vel_z` (-1.0): z軸速度へのペナルティ
   - `base_height` (-10.0): 高さ維持
   - `action_rate` (-0.01): アクション変化のペナルティ
   - `similar_to_default` (-0.05): デフォルト姿勢維持

4. **終了条件**:
   - エピソード長が最大値（1000ステップ = 20秒）に達した
   - ロボットが転倒（roll/pitch > 30度）
   - 追跡者が0.5m以内に接近（捕まった）

### 追跡者の行動 (`_update_chaser_behavior`)

- 逃走者の位置を常に追跡
- 速度: 0.6 m/s（設定可能）
- 簡単な歩行パターンで移動
- より高度な追跡が必要な場合は、追跡者も学習させることが可能

## 使い方

### 1. 訓練

```bash
# 基本的な訓練（4096環境、500イテレーション）
python go2_tag_train.py --exp_name go2-tag-game --num_envs 4096 --max_iterations 500

# デバッグ用（ビューアーを表示）
python go2_tag_train.py --exp_name go2-tag-game --num_envs 16 --max_iterations 10 --show_viewer

# 事前学習済みモデルから継続訓練
python go2_tag_train.py --exp_name go2-tag-game-v2 --load_exp_name go2-tag-game --ckpt 500
```

### 2. 評価

```bash
# 学習済みモデルで評価（ビューアー付き）
python go2_tag_eval.py --exp_name go2-tag-game --ckpt 500
```

## パラメータ調整

### 難易度調整

`go2_tag_train.py`の`get_cfgs()`関数で以下のパラメータを調整できます：

```python
env_cfg = {
    "chaser_speed": 0.6,  # 追跡者の速度（大きいほど難しい）
    "caught_distance": 0.5,  # 捕まる距離（小さいほど難しい）
    "chaser_init_pos": [-3.0, 0.0, 0.42],  # 追跡者の初期位置
}

reward_cfg = {
    "avoid_chaser": 5.0,  # 逃げる報酬（大きいほど逃走を優先）
    "forward_vel": 1.0,  # 前進報酬（大きいほど積極的に動く）
    "survival": 0.5,  # 生存報酬
}
```

## トラブルシューティング

### Q: 学習が進まない
A: 以下を試してください：
- `chaser_speed`を下げる（0.4-0.5）
- `avoid_chaser`報酬を上げる（10.0）
- `num_envs`を増やす（8192）

### Q: ロボットがすぐに転倒する
A: 以下を試してください：
- `termination_if_roll_greater_than`を増やす（45度）
- `base_height`ペナルティを増やす（-20.0）

### Q: 追跡者の動きがおかしい
A: `_update_chaser_behavior()`の歩行パターンを調整するか、事前学習済みの歩行ポリシーを使用してください。
